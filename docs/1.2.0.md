# NeoTab 1.2.0

`1.2.0` starts the ingame configuration foundation for NeoTab. It keeps the existing command and config behavior and adds a small GUI layer on top.

## Compatibility

- Platform: Paper
- Minecraft target: `1.20.6+`
- Build API: Paper `1.21-R0.1-SNAPSHOT`
- Java bytecode: Java 21
- PlaceholderAPI: optional
- LuckPerms: optional

## Control Panel

Open the GUI:

```text
/tab gui
```

Permission:

```text
neotab.gui
```

The first GUI has three categories:

- `Tab`
- `Scoreboard`
- `Extras`

The GUI uses normal Bukkit inventories and cancels clicks/drags so players cannot take GUI items.

## Tab Menu

The `Tab` menu currently supports:

- `Name`: starts chat input for the tab/header name.
- `Style`: opens a selection menu for the existing animation styles.
- `Back`: returns to the main menu.

Chat input:

- Type the new value in chat.
- Type `cancel` to cancel.
- The value is applied live and saved to `config.yml`.

Existing commands still work:

```text
/tab setname <name>
/tab style <rainbow|purple-pulse|gradient-wave|static>
```

## Scoreboard

The scoreboard system is intentionally minimal in this first pass.

Config:

```yaml
scoreboard:
  enabled: false
  update-interval-ticks: 20
  title: "<gradient:#00ffaa:#0088ff>NeoTab</gradient>"
  lines:
    - "&7Online: &b{online}/{max}"
    - "&7Ping: &e{ping}ms"
    - "&7RAM: &a{ram_used}/{ram_max} MB"
  presets: {}
```

Commands:

```text
/tab sb on
/tab sb off
/tab sb toggle
/tab sb title <text>
/tab sb line <1-15> <text>
/tab sb clear <1-15>
/tab sb clearall
/tab sb save <name>
/tab sb load <name>
/tab sb list
```

Permissions:

```text
neotab.scoreboard
neotab.scoreboard.toggle
neotab.scoreboard.edit
neotab.scoreboard.presets
```

Supported built-in placeholders:

```text
{online}
{max}
{ping}
{avg_ping}
{ram_used}
{ram_max}
{ram_percent}
{server_name}
```

PlaceholderAPI placeholders in scoreboard title and lines are supported when PlaceholderAPI is installed and `enable-placeholderapi` is true.

## Extras

The `Extras` menu currently supports:

- `Interval`: uses the existing performance presets (`smooth`, `balanced`, `light`).
- `ActionBar Timer`: opens simple timer controls.

No new performance system was added. The GUI calls the existing performance config flow.

## ActionBar Timer

Config:

```yaml
extras:
  actionbar-timer:
    enabled: true
```

Commands:

```text
/tab timer start <duration>
/tab timer stop
/tab timer pause
/tab timer resume
```

Duration examples:

```text
30s
5m
10m
1h
```

Permissions:

```text
neotab.timer
neotab.timer.admin
```

The timer is per-player, updates once per second, uses the ActionBar, and is cleaned up when the player leaves.

## Intentionally Minimal

Not included in this first `1.2.0` foundation:

- BossBar timer
- Drag-and-drop GUI editing
- Color picker
- Web dashboard
- Database storage
- Legacy support below `1.20.6`
- Large adapter rewrite
