# NeoTab 1.1.0 Beta.2

`1.1.0-Beta.2` is the second beta release after the original `1.0.2` JAR was recovered and rebuilt as source.

## Compatibility

- Platform: Paper
- Minecraft target: `1.20.6+`
- Build API: Paper `1.21-R0.1-SNAPSHOT`
- Plugin API version: `1.20`
- Java bytecode: Java 21
- Tested with PlaceholderAPI `2.12.2`

The Modrinth project currently lists compatibility around `1.21.4-1.21.11` and `26.1.x`.

## What Changed Since 1.0.2

- Added optional PlaceholderAPI support
- Added `enable-placeholderapi` config option
- PlaceholderAPI placeholders work in `server-name`
- PlaceholderAPI placeholders work in `ram-format`
- Build target moved to Paper `1.21`
- Java bytecode target moved to Java 21
- Source project, Gradle build, and GitHub releases were restored
- Fixed loading on Paper `1.20.6` by lowering `api-version` to `1.20`
- Fixed optional LuckPerms handling when LuckPerms is not installed

## PlaceholderAPI

PlaceholderAPI is optional. If it is installed, NeoTab loads it through `softdepend`.

Enable it in `config.yml`:

```yaml
enable-placeholderapi: true
```

Example:

```yaml
server-name: "<gradient:#AA00AA:#BA55D3>NeoTab: %player_name%</gradient>"
ram-format: "<gray>PAPI: <light_purple>%player_name%</light_purple> | RAM: <light_purple>{used}MB / {total}MB</light_purple> | Ping: {playerPing}ms</gray>"
```

Some PlaceholderAPI placeholders require expansions. For example, `%player_name%` needs the Player expansion.

## Commands

```text
/tab reload
/tab setname <name>
/tab style <rainbow|purple-pulse|gradient-wave|static>
```

## Known Beta Notes

- Header animation currently recolors the plain text form of `server-name`.
- PlaceholderAPI values in the header work, but complex MiniMessage formatting returned by placeholders may be flattened by the animation.
- The footer still uses a legacy compatibility path internally and should be cleaned up before a final stable `1.1.0`.
