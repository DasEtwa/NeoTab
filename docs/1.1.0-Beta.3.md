# NeoTab 1.1.0 Beta.3

`1.1.0-Beta.3` is the third beta release after the original `1.0.2` JAR was recovered and rebuilt as source.

## Compatibility

- Platform: Paper
- Minecraft target: `1.20.6+`
- Build API: Paper `1.21-R0.1-SNAPSHOT`
- Plugin API version: `1.20`
- Java bytecode: Java 21
- Tested with PlaceholderAPI `2.12.2`

The Modrinth project currently lists compatibility around `1.21.4-1.21.11` and `26.1.x`.

## What Changed Since 1.0.2

- Added optional PlaceholderAPI support
- Added `enable-placeholderapi` config option
- PlaceholderAPI placeholders work in `server-name`
- PlaceholderAPI placeholders work in `ram-format`
- Build target moved to Paper `1.21`
- Java bytecode target moved to Java 21
- Source project, Gradle build, and GitHub releases were restored
- Fixed loading on Paper `1.20.6` by lowering `api-version` to `1.20`
- Fixed optional LuckPerms handling when LuckPerms is not installed
- Added a safe Modrinth update checker
- Removed forced bold formatting from animated headers by default
- Footer rendering now sends Adventure components directly instead of using a manual legacy fallback
- LuckPerms prefix/suffix metadata keeps MiniMessage and legacy color formatting where possible
- Added in-game performance presets for the tab update interval

## PlaceholderAPI

PlaceholderAPI is optional. If it is installed, NeoTab loads it through `softdepend`.

Enable it in `config.yml`:

```yaml
enable-placeholderapi: true
```

Example:

```yaml
server-name: "<gradient:#AA00AA:#BA55D3>NeoTab: %player_name%</gradient>"
ram-format: "<gray>PAPI: <light_purple>%player_name%</light_purple> | RAM: <light_purple>{used}MB / {total}MB</light_purple> | Ping: {playerPing}ms</gray>"
```

Some PlaceholderAPI placeholders require expansions. For example, `%player_name%` needs the Player expansion.

## Update Checker

NeoTab checks Modrinth asynchronously after startup when enabled:

```yaml
update-checker:
  enabled: true
  include-beta: false
  notify-admins: true
  check-delay-seconds: 5
```

Behavior:

- Uses the Modrinth project slug `neotab`.
- Checks only Paper-compatible versions for the current server Minecraft version.
- Does not auto-download or install updates.
- Logs a concise console message when an update is available.
- Notifies online players with `neotab.update.notify` if `notify-admins` is enabled.
- `include-beta: false` only considers release versions.
- `include-beta: true` also considers Modrinth beta versions.

## Header Formatting

Animated headers no longer force bold text. To restore the old look:

```yaml
header:
  bold-animation: true
```

## Performance Presets

Admins can change the tab update interval without editing files:

```text
/tab performance smooth
/tab performance balanced
/tab performance light
/tab performance custom 10
/tab performance save event
```

Default intervals:

```yaml
performance:
  active-preset: "smooth"
  presets:
    smooth: 3
    balanced: 10
    light: 20
  saved-presets: {}
update-interval-ticks: 3
```

Behavior:

- `smooth` updates every 3 ticks.
- `balanced` updates every 10 ticks.
- `light` updates every 20 ticks.
- `custom [ticks]` accepts values from `1` to `1200`.
- Every performance command saves the active setting to `config.yml`.
- `save [name]` stores the current interval in `performance.saved-presets`.
- Saved presets can be used again with `/tab performance [name]`.
- Permission: `neotab.performance`.

## Known Beta Notes

- Header animation currently recolors the plain text form of `server-name`.
- PlaceholderAPI values in the header work, but complex MiniMessage formatting returned by placeholders may be flattened by the animation.
