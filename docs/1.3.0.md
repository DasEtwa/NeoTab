# NeoTab 1.3.0 Beta

`1.3.0-Beta.3` continues the next development cycle after the stable `1.2.0` release.

## Theme

ActionBar Extras.

This beta is not the large `2.0` profiles and conditions rewrite. It builds on the existing `1.2.0` GUI, Scoreboard, Extras, and ActionBar Timer systems.

## Highlights

- Central ActionBar priority dispatcher.
- Existing ActionBar Timer now participates in the priority system.
- New per-player Stopwatch.
- Configurable real-time Clock module.
- Configurable Welcome ActionBar after join.
- Random ActionBar messages.
- Biome Popup when a player enters a new biome.
- Minecraft Advancement counter exposed as Achievements.
- Performance Notice GUI category for heavier modules.
- Nearest Player module with same-world and max-distance controls.
- Structure Popup config and GUI placeholder, intentionally no structure lookup task yet.
- Region Profiles for automatic tab and scoreboard profile switching in configured cuboid regions.
- Bugfix pass for global ActionBar disable handling, scoreboard interop safety, UTF-8/default docs cleanup, expanded command usage text, and compile-time protection for the documented `1.20.6+` support range.

## Priority Order

| Priority | Module |
| --- | --- |
| `100` | Timer |
| `100` | Stopwatch |
| `90` | Biome Popup |
| `85` | Structure Popup reserved |
| `70` | Welcome |
| `50` | Nearest Player |
| `40` | Achievements |
| `30` | Clock |
| `10` | Random Messages |

Expired messages are ignored and removed. The dispatcher sends the highest-priority active message per player.

Existing timer commands remain:

```text
/tab timer start <duration>
/tab timer stop
/tab timer pause
/tab timer resume
/tab timer text <text with {time}>
```

If a timer starts while a stopwatch is running, NeoTab stops the stopwatch for that player. If a stopwatch starts while a timer is running, NeoTab rejects the stopwatch start.

The old `extras.actionbar-timer` config path from `1.2.0` is still read when the new `extras.actionbar.timer` path is missing.

Region data is stored in `regions.yml`:

```yaml
regions:
  spawn:
    enabled: true
    world: "world"
    priority: 100
    min:
      x: -50
      y: 60
      z: -50
    max:
      x: 50
      y: 120
      z: 50
    tab-profile: "spawn"
    scoreboard-profile: "spawn"
```

`default` profile names use the normal global tab and scoreboard config. Named tab profiles are read from `tab-profiles` in `config.yml`; named scoreboard profiles reuse `scoreboard.presets`.

`/tab region gui` opens a simple region editor. It lists regions, opens an edit view on click, can set positions from your current location, adjust priority, change profile names through chat input, toggle regions, delete with confirmation, create from the NeoTab wand selection, and import a WorldEdit/FAWE selection when available.

## Performance Safety

- All Bukkit and Paper API reads stay on the main server thread.
- No async PlaceholderAPI calls are introduced.
- Clock, Random Messages, and Achievements have a 60 second minimum interval.
- Achievements count visible Minecraft advancements only, so recipes and hidden internal advancements are ignored.
- Biome Popup has a 30 tick minimum interval.
- Nearest Player has a 40 tick minimum interval and uses squared distance for comparisons.
- Structure Popup has no detection task in this beta.
- Region checks run only when a player changes block position or world, cache the current winning region per player, and filter regions by world before checking bounds.

## Compatibility Target

- Platform: Paper
- Minecraft target: `1.20.6+`
- Build API: Paper `1.20.6-R0.1-SNAPSHOT`
- Java bytecode: Java 21
- PlaceholderAPI: optional
- LuckPerms: optional

## Not Included

- No profiles or conditions system.
- No per-world UI profiles.
- No plugin hooks.
- No database.
- No web dashboard.
- No BossBar system.
- No drag-and-drop editor.
- No support below Minecraft/Paper `1.20.6`.
