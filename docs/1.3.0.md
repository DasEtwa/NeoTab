# NeoTab 1.3.0 Beta

`1.3.0-Beta.1` starts the next development cycle after the stable `1.2.0` release.

## Theme

ActionBar Extras.

This beta is not the large `2.0` profiles and conditions rewrite. It builds on the existing `1.2.0` GUI, Scoreboard, Extras, and ActionBar Timer systems.

## Highlights

- Central ActionBar priority dispatcher.
- Existing ActionBar Timer now participates in the priority system.
- New per-player Stopwatch.
- Configurable real-time Clock module.
- Configurable Welcome ActionBar after join.
- Random ActionBar messages.
- Biome Popup when a player enters a new biome.
- Minecraft Advancement counter exposed as Achievements.
- Performance Notice GUI category for heavier modules.
- Nearest Player module with same-world and max-distance controls.
- Structure Popup config and GUI placeholder, intentionally no structure lookup task yet.

## Priority Order

| Priority | Module |
| --- | --- |
| `100` | Timer |
| `100` | Stopwatch |
| `90` | Biome Popup |
| `85` | Structure Popup reserved |
| `70` | Welcome |
| `50` | Nearest Player |
| `40` | Achievements |
| `30` | Clock |
| `10` | Random Messages |

Expired messages are ignored and removed. The dispatcher sends the highest-priority active message per player.

## Commands

```text
/tab stopwatch start
/tab stopwatch stop
/tab stopwatch pause
/tab stopwatch resume
/tab stopwatch reset
/tab clock on
/tab clock off
/tab clock timezone <zone>
/tab clock format <format>
/tab welcome on|off
/tab randommessages on|off
/tab biomepopup on|off
/tab nearestplayer on|off
/tab achievements on|off
```

Existing timer commands remain:

```text
/tab timer start <duration>
/tab timer stop
/tab timer pause
/tab timer resume
/tab timer text <text with {time}>
```

If a timer starts while a stopwatch is running, NeoTab stops the stopwatch for that player. If a stopwatch starts while a timer is running, NeoTab rejects the stopwatch start.

## GUI

```text
/tab gui
-> Extras
   -> ActionBar
      -> Timer
      -> Stopwatch
      -> Clock
      -> Welcome
      -> Random Messages
      -> Biome Popup
      -> Achievements
      -> Performance Notice
         -> Nearest Player
         -> Structure Popup
```

The GUI stays minimal: toggles and simple controls only. Text editing still uses the existing chat input where already supported.

## Config

The new config lives under:

```yaml
extras:
  actionbar:
    enabled: true
    timer:
      enabled: true
      running-format: "{time}"
      paused-format: "Paused {time}"
      ended-format: "timer ends"
    stopwatch:
      enabled: true
      text: "<light_purple>Stopwatch {time}</light_purple>"
    clock:
      enabled: false
      timezone: "Europe/Berlin"
      interval-seconds: 60
      min-interval-seconds: 60
      format: "HH:mm"
      text: "<gray>Time: <light_purple>{time}</light_purple></gray>"
    welcome:
      enabled: true
      delay-ticks: 20
      duration-seconds: 5
      text: "<gradient:#AA00AA:#BA55D3>Welcome {player}!</gradient>"
    random-messages:
      enabled: false
      interval-seconds: 300
      min-interval-seconds: 60
      duration-seconds: 5
      messages:
        - "<light_purple>Drink water! <3</light_purple>"
        - "<gray>Have a nice day!</gray>"
        - "<gradient:#AA00AA:#BA55D3>Thanks for playing!</gradient>"
    biome-popup:
      enabled: false
      check-interval-ticks: 40
      min-check-interval-ticks: 30
      duration-seconds: 7
      text: "<gray>Entering <light_purple>{biome}</light_purple></gray>"
    achievements:
      enabled: false
      provider: "minecraft"
      interval-seconds: 60
      min-interval-seconds: 60
      text: "<gray>Achievements: <light_purple>{completed}</light_purple>/<light_purple>{total}</light_purple></gray>"
    nearest-player:
      enabled: false
      check-interval-ticks: 60
      min-check-interval-ticks: 40
      max-distance: 100
      same-world-only: true
      text: "<gray>Nearest: <light_purple>{player}</light_purple> <gray>({distance} blocks)</gray>"
    structure-popup:
      enabled: false
      experimental: true
      check-interval-ticks: 200
      min-check-interval-ticks: 100
      max-distance: 64
      duration-seconds: 7
      text: "<gray>Nearby structure: <light_purple>{structure}</light_purple></gray>"
```

The old `extras.actionbar-timer` config path from `1.2.0` is still read when the new `extras.actionbar.timer` path is missing.

## Performance Safety

- All Bukkit and Paper API reads stay on the main server thread.
- No async PlaceholderAPI calls are introduced.
- Clock, Random Messages, and Achievements have a 60 second minimum interval.
- Biome Popup has a 30 tick minimum interval.
- Nearest Player has a 40 tick minimum interval and uses squared distance for comparisons.
- Structure Popup has no detection task in this beta.

## Compatibility Target

- Platform: Paper
- Minecraft target: `1.20.6+`
- Java bytecode: Java 21
- PlaceholderAPI: optional
- LuckPerms: optional

## Not Included

- No profiles or conditions system.
- No per-world UI profiles.
- No plugin hooks.
- No database.
- No web dashboard.
- No BossBar system.
- No drag-and-drop editor.
- No support below Minecraft/Paper `1.20.6`.
